# 🌾 SAGRI Project Report: Smart Agriculture Real-time Intelligence

## 1. Executive Summary
**SAGRI (Krishi Sahayak)** is a next-generation AI platform designed to bridge the gap between traditional farming and modern data science. By providing real-time insights into crop health, market trends, and environmental risks, SAGRI empowers farmers to make data-driven decisions that increase yield and minimize loss.

## 2. Technical Architecture & Tech Stack

### High-Level Architecture
SAGRI follows a modern **Decoupled Architecture**:
-   **Frontend**: A responsive, interactive Single Page Application (SPA).
-   **Backend**: A high-performance Python-based API server.
-   **AI Layer**: Specialized machine learning models integrated into the backend.
-   **Database/Backend-as-a-Service**: Leveraging Supabase for real-time data and authentication.

## 4. Machine Learning Ecosystem

### 🔬 Crop Disease Detection
-   **Model**: Transfer learning with **MobileNetV2** (Pre-trained on ImageNet).
-   **Accuracy**: High-precision classification of plant leaf images.
-   **Dataset**: Augmented "PlantVillage" dataset with 38+ disease classes.
-   **Output**: Returns disease name, confidence level, severity, and immediate treatment steps.

### 📈 Market Price Forecasting
-   **Model**: **Facebook Prophet** (Time-series analysis).
-   **Functionality**: Predicts future prices for major crops (Wheat, Rice, Cotton, etc.) up to 30 days ahead based on historical trends.

### 🌱 Smart Crop Recommendation
-   **Model**: **Random Forest Classifier**.
-   **Input Parameters**: Nitrogen (N), Phosphorus (P), Potassium (K), Temperature, Humidity, Soil pH, and Rainfall.
-   **Goal**: Recommends the most suitable crop for specific soil conditions and climate.

### ⚠️ Crop Risk Prediction
-   **Model**: **Random Forest**.
-   **Focus**: Analyzes environmental conditions to calculate the probability of crop failure or high-risk events.

## 6. Installation & Deployment

### Prerequisites
-   **Node.js** (v18 or higher)
-   **Python** (v3.10 or higher)

### Automated Setup
The project includes a `setup_all.py` script that handles backend environment creation, dependency installation, and frontend setup in one command:
```bash
python setup_all.py
```

### Manual Configuration
1.  **Backend**: Navigate to `/backend`, create a virtual environment, and run `pip install -r requirements.txt`. Start with `uvicorn main:app --reload`.
2.  **Frontend**: Run `npm install` followed by `npm run dev` in the root directory.
